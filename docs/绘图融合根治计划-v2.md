# 绘图融合根治计划 v2

> 状态：执行中 · 决策：HOLD · 目标：完成本计划后才允许宣告“根治”

## 为什么重开一轮

第一轮 LE-001～LE-007 建立了 React Flow 单相机、committed SVG 世界和局部 Excalidraw 编辑事务，并证实普通平移/缩放时视口漂移为 0。

但 2026-07-15 三路独立 Review 证明旧验收漏掉了以下反例：

- SVG 上一帧仍在屏幕上时，命中已经读取新 `canvas.drawing`；
- cold start 首个 snapshot 为空，300/800 元素首次显现约 229/238ms；
- hole-punch 先删元素再每 48 个分组，编辑前部元素会使后续所有组重导；
- 局部事务合并会把非连续的关系闭包强制插回一个连续区间，改变无关元素 z-order；
- 编辑相机恢复期间 live editor `pointer-events:none`，输入可穿透到底下节点；
- 容器承载只在 drag stop 后追赶，不是拖动中同相位运动；
- 编辑期间没有可靠草稿保存；elements/files 的服务端落盘也不是一个故障原子单元；
- 旧 4518 红线在 paint 前结束计时，且 Vite `/@fs` 隔离边界过宽。

因此第一轮证据作为历史保留，不得用来替代本轮证伪。

## 根治后的六条不变量

1. **帧真相唯一**：像素、命中、小地图和可见性决策只能读取最后已 paint 的 rendered revision。
2. **相机与输入主权唯一**：手势全程只有 React Flow viewport；编辑器隐藏时必须仍有同层输入盾，不得穿透。
3. **局部性可证伪**：hole-punch 不改变未受影响组的槽位；编辑关系闭包不改变无关元素的全局顺序。
4. **容器承载同相位**：拖动中容器、成员和锚定墨迹使用同一个临时位移；松手只做一次持久化收口。
5. **草稿可恢复**：正常编辑定期产生本地草稿日志；刷新/崩溃后可恢复；`pagehide` 只是补充而不是唯一保存策略。
6. **持久化引用一致**：任何已落盘 element 引用的图片必须已存在；任意中间步失败都不得产生缺资产引用。

## 验收红线

- `npm test` 全绿，且每条 P1 至少有一个先红后绿的回归。
- `npm run build` 全绿，主包不得静态引入 subset worker core。
- 4518 不得代理 API，不得通过 `/@fs` 读取 `data/`，不得访问真实持久化端点。
- 300/800 元素 cold 的可见占位反馈 <100ms；完整绘图 paint 实测如超 100ms，必须保证无空帧/无隐形热区并记录确切数字，不得用 500ms 冒充及格。
- 相机平移/缩放 viewport drift max = 0；尾帧期间底下节点 pointer 计数 = 0。
- 反复打开/关闭首组元素时，非目标组 export 计数 = 0。
- 容器拖动各采样帧的容器/墨迹相对位移 = 0。
- 实机 4517 必须 Cmd+R 后检查 console 无 TDZ/worker/运行时错误，但严禁自动坐标拖拽真画布。

## 数据与操作边界

- 珍贵数据：`data/{canvas,layout,enrich,drawing-files}.json`；本计划不使用它们作为自动化 fixture。
- 修改 `web/` 后直接 build；修改 `server/` 后只有 `launchctl kickstart -k gui/$(id -u)/com.bingo.agent-canvas` 能让真页运行新代码，执行前必须再获得用户确认。
- 不修改 schema，不删除历史 Loop 证据，不 push。
